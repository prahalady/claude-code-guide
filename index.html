기본 규칙
정해주기

가장 중요한 포스트잇 메모 · 핵심 업무 수칙

새로 온 비서의 모니터에 가장 중요한 포스트잇 메모를 붙여주는 단계입니다. 비서가 일을 시작할 때마다 항상 가장 먼저 쳐다보는 핵심 업무 수칙을 여기에 담습니다.

프로젝트의 기본 규칙 — 어떤 언어를 쓸지, 어떤 도구를 선호하는지, 절대 잊으면 안 되는 컨벤션 — 을 간결하게 적어두면 Claude는 매 대화마다 이 규칙을 가장 먼저 참조합니다.

비유

"문서를 작성할 때는 항상 존댓말을 써라", "우리 팀은 A 프로그램 대신 항상 B 프로그램을 쓰니까 명심해라"와 같이, 비서가 매 순간 절대 잊으면 안 되는 프로젝트의 기본 규칙을 적어둡니다.

STEP 02

매뉴얼
쥐여주기

재사용 가능한 작업 지침서 · 서랍 속 매뉴얼

비서에게 회사의 모든 지식을 다 외우라고 강요할 수는 없습니다. 대신, 특정 업무를 할 때만 꺼내서 읽어볼 수 있는 재사용 가능한 작업 지침서(매뉴얼)를 책상 서랍에 넣어줍니다.

스킬은 특정 작업 유형에 특화된 지침, 예시, 베스트 프랙티스를 담은 파일 묶음입니다. 평소에는 메모리에 올라가지 않다가, 해당 업무가 필요할 때 Claude가 알아서 꺼내 읽고 따릅니다.

비유

평소에는 잊고 있다가, 상사가 "웹사이트에 배포해 줘"라고 지시하면 비서가 알아서 '웹사이트 배포 매뉴얼'을 서랍에서 꺼내 순서대로 따라 하는 식입니다.

03
MCP
STEP 03

외부 전화기
놓아주기

Model Context Protocol · 외부 서비스 연결

AI 비서는 원래 자기 컴퓨터(두뇌) 안에서만 일할 수 있습니다. MCP는 외부 세상과 소통할 수 있도록 만능 사내 전화기를 놔주는 단계입니다.

MCP(Model Context Protocol)는 Claude가 외부 데이터베이스, 사내 메신저, 파일 시스템, 웹 서비스 등 다양한 외부 도구와 직접 연결할 수 있게 해주는 표준 프로토콜입니다.

비유

"고객 데이터베이스에서 홍길동 님의 자료를 찾아줘", "사내 메신저(Slack)로 마케팅팀에 회의 끝났다고 알림 좀 보내줘"라고 지시하면, 비서가 MCP라는 전화기를 써서 외부 서비스에 직접 접속하고 소통합니다.

인턴
채용하기

서브에이전트 · 자료 조사 전문 인턴

비서에게 한 번에 100쪽짜리 문서 10개를 읽으라고 하면, 비서도 머리가 복잡해져서 중요한 내용을 까먹습니다. 그래서 메인 비서를 도울 따로 파견 보내는 요약 전문 인턴을 하나 더 부릅니다.

서브에이전트는 메인 Claude와 완전히 분리된 컨텍스트 창을 가진 별도의 AI 인스턴스입니다. 방대한 자료를 독립적으로 처리하고 핵심 요약만 메인 에이전트에게 전달하여 전체 시스템의 효율을 극대화합니다.

비유

방대한 시장 조사 자료를 분석할 때, 메인 비서가 직접 읽는 대신 서브에이전트(인턴)를 별개의 방으로 보냅니다. 인턴이 방대한 자료를 다 읽고 핵심 요약본만 딱 한 장으로 정리해서 메인 비서에게 가져다주어 효율을 높입니다.

STEP 05

자동 스위치
달기

훅 · 기계식 자동화 트리거

비서가 굳이 똑똑한 두뇌(AI)를 쓰지 않아도 알아서 돌아가게 만드는 기계적인 자동 스위치를 설치합니다. 특정 이벤트가 발생하면 무조건 실행되는 자동화 규칙입니다.

훅은 Claude의 AI 판단을 거치지 않고 특정 행동(파일 저장, 명령 실행 전후 등)에 자동으로 반응하는 스크립트입니다. 코드 품질 검사, 포맷팅, 로깅 등 반복적인 작업에 이상적입니다.

비유

"내가 작성한 코드를 저장할 때마다 무조건 맞춤법/코드 검사기(ESLint)를 돌려라"처럼, 특정 행동을 하면 무조건 기계적으로 작동하는 자동화 세팅입니다.

→

→
자동 테스트 실행 & 로그 기록

STEP 06

공구 세트로
포장하기

플러그인 · 종합 도구 패키지

앞에서 만든 매뉴얼(스킬), 전화기(MCP), 인턴(서브에이전트), 자동 스위치(훅)를 하나의 박스에 예쁘게 포장해 종합 공구 세트로 만드는 단계입니다.

플러그인은 여러 도구와 설정을 하나의 패키지로 묶어 다른 프로젝트에 즉시 적용하거나 팀원 및 커뮤니티와 공유할 수 있게 해줍니다. 한 번 잘 만들어두면 어디서든 재사용이 가능합니다.

비유

내가 완벽하게 세팅한 '신입 비서 업무 도구 박스'를 만들어서 다른 팀 프로젝트에 그대로 복사해서 쓰거나, 마켓플레이스를 통해 다른 회사 사람들에게 공유(배포)할 수 있습니다.

FINAL STEP

비서의 뇌 용량
관리하기

맥락 비용 (Context Cost) · 가장 중요한 마지막 단계

이 도구들이 좋다고 해서 모니터에 포스트잇을 100장 붙이고(CLAUDE.md), 매뉴얼을 수십 개씩 꺼내두면(스킬), 비서가 혼란에 빠져 엉뚱한 실수를 하게 됩니다. 각 도구가 가장 잘하는 역할만 딱딱 맡겨서 뇌 용량을 아껴주는 것이 핵심입니다.

과부하 상태 (잘못된 운영)
CLAUDE.md 100줄 + 스킬 30개 동시 로드 + 무분별한 MCP → 오류 급증
최적 상태 (올바른 운영)
핵심 규칙만 CLAUDE.md + 필요한 스킬만 호출 + 서브에이전트 위임 → 안정적 성능
올바른 방법 01
CLAUDE.md
간결하게

항상 지켜야 할 기본 규칙만 짧고 명확하게 적어둡니다. 길고 복잡한 규칙은 오히려 성능을 저하시킵니다.

올바른 방법 02
무거운 작업은
인턴에게

방대한 자료를 읽는 일은 메인 비서의 뇌를 쓰지 않도록 서브에이전트(인턴)에게 넘깁니다.

올바른 방법 03
도구는
필요할 때만

외부 연락이 필요할 때만 MCP와 스킬을 결합해 잠깐 꺼내 씁니다. 항상 켜두는 것은 낭비입니다.
